# EdgeX 工业网关 UI 设计规范

> 基于 **Arco Design Vue** 组件库，面向工业网关场景的现代化清爽界面规范。  
> 技术栈：Vue 3 · Arco Design · Tailwind CSS（辅助）· CSS Variables

---

## 1. 设计理念

### 1.1 风格定位：现代工业清爽

在保留工业软件**信息密度高、操作路径短**特点的同时，引入当代 SaaS 产品的视觉语言：

| 维度 | 旧版（线构工业） | 新版（现代清爽） |
|------|-----------------|-----------------|
| 圆角 | 零圆角 | 轻圆角 `4–8px`，卡片/输入框 `6px` |
| 边框 | 粗重 1px 实线堆叠 | 细线 `#e2e8f0`，减少双边框 |
| 阴影 | 几乎不用 | 极轻阴影，仅用于浮层/下拉 |
| 色彩 | 深灰 `#0f172a` 主色 | Slate 中性色 + Sky 功能色 |
| 留白 | 紧凑 | 适度放大间距，提升呼吸感 |
| 动效 | 无 | 150–200ms 过渡，克制使用 |

### 1.2 设计原则

- **层次靠间距，不靠装饰**：用 8px 栅格与字重区分层级，避免过多分割线与色块
- **功能色只用于状态**：成功/警告/错误/协议标签，正文与背景保持中性
- **一屏一事**：列表页工具栏 + 内容区结构统一，减少视觉噪音
- **深浅主题对等**：所有组件必须同时适配 Light / Dark，禁止硬编码 `#fff`
- **可扫描优先**：表格、监控数据支持快速扫读；长文本悬停展示完整内容
- **同区同尺寸**：同一表单区域 / 面板内，所有输入类控件必须使用相同 `size`，禁止混用

## 2. 设计令牌（Design Tokens）

全局 CSS 变量定义于 `ui/src/App.vue` 与 `ui/src/styles/globals.css`，页面级样式优先引用变量。

| 层级 | 字号 | 字重 | 行高 | 场景 |

### 2.3 间距（8px 栅格）

| Token | 值 | 用途 |
|-------|-----|------|
| `--space-1` | 4px | 图标与文字间距 |
| `--space-2` | 8px | 紧凑元素间距、按钮组 |
| `--space-3` | 12px | 表单项之间、字段堆叠间距 |
| `--space-4` | 16px | 区块内边距、行间距 |
| `--space-5` | 20px | 区块间距 |
| `--space-6` | 24px | 页面内边距、大卡片 |
| `--space-8` | 32px | 页面级区块分隔 |

#### 间距组合速查

| 场景 | 标签 → 控件 | 控件 → 控件 | 控件 → 说明/错误 |
|------|------------|------------|-----------------|
| 标准表单 | 6px (`gap`) | 12px (`field-stack`) | 4px |
| 弹窗顶栏表单 | 6px | 16px (`mb-4` 行间距) | 4px |
| 面板内筛选区 | 6px | 12px | — |
| 表格工具栏 | — | 8px（按钮组） | — |

**禁止**：相邻输入控件 margin 为 0 或直接贴边；说明文字与下一字段间距 < 8px。

### 2.4 圆角与阴影

```css
--radius-sm: 4px;   /* Tag、小 Badge */
--radius-md: 6px;   /* 输入框、按钮、表格容器 */
--radius-lg: 8px;   /* 卡片、弹窗、Banner */
--radius-full: 9999px; /* 状态圆点、头像 */

--shadow-sm: 0 1px 2px rgba(15, 23, 42, 0.05);
--shadow-md: 0 4px 12px rgba(15, 23, 42, 0.08);  /* 下拉菜单、Popover */
--shadow-none: none;  /* 表格、列表默认 */
```

**规则**：表格主体、数据列表不使用阴影；浮层（Dropdown、Modal、Notification）可使用 `--shadow-md`。

### 2.5 组件尺寸体系（Arco Size）

Arco Design 通过 `size` 属性控制控件高度。**同一视觉区域必须统一 size，不可混用。**

| Arco `size` | 控件高度 | 字号 | 使用场景 |
|-------------|---------|------|----------|
| `large` | 36px | 14px | 登录页、安装向导主输入 |
| **默认 / 省略** | **32px** | **14px** | **页面表单、弹窗、筛选区（推荐默认）** |
| `small` | 28px | 13px | 表格内嵌编辑、紧凑积木块（整组使用） |
| `mini` | 24px | 12px | 表格行内图标按钮、Radio 组、Tag 操作 |

#### 尺寸选用决策

```
页面级表单 / 弹窗 / Modal
  └─ 默认（32px）+ class="form-controls-md"

表格单元格内编辑
  └─ size="small"（28px）+ class="form-controls-sm"

行内操作（↑↓删除、模式切换）
  └─ size="mini"（24px）— 非输入框，可与表单混排
```

#### 同类控件高度对齐

| 控件组合 | 要求 |
|----------|------|
| `a-input` + `a-select` | 必须相同 `size` |
| `a-input-search` 输入框 + 搜索按钮 | 按钮高度 = 输入框高度（由 `form-controls-md` 保证） |
| `a-input` + `a-button`（同行） | 按钮使用与输入相同 size 或 `size="small"` 搭配 28px 整行 |
| `a-switch` + 标签 | Switch 独立对齐，不与输入框比高度 |
| `a-textarea` | 不设固定高度，用 `auto-size`；**字号与同级 input 一致** |

#### 反模式（禁止）

```vue
<!-- ❌ 同面板混用 default 与 small -->
<a-select size="small" />
<a-input />  <!-- default 32px，与上方 28px 不一致 -->

<!-- ❌ 写了 mb-2 但未定义 CSS 类 -->
<a-input class="mb-2" />

<!-- ✅ 正确：统一尺寸 + 字段堆叠 -->
<div class="form-controls-md field-stack">
  <div class="form-field">
    <div class="field-label-sm">源设备通道</div>
    <a-select>...</a-select>
  </div>
  <div class="form-field">
    <div class="field-label-sm">筛选设备</div>
    <a-input-search search-button />
  </div>
</div>
```

## 3. 布局结构

### 3.1 应用骨架

```
┌─────────────┬──────────────────────────────────┐
│  Sidebar    │  Header（面包屑 + 主题 + 用户）    │
│  220px      ├──────────────────────────────────┤
│  (可折叠64) │  Main Content                     │
│             │  padding: 24px                    │
└─────────────┴──────────────────────────────────┘
```

- **侧栏** `.industrial-sidebar`：宽 220px，折叠后 64px；导航项高 40px，激活态左侧 2px Primary 色条
- **顶栏** `.industrial-header`：高 56px，背景 `--surface-raised`，底边框 1px
- **内容区** `.page-container`：最大宽度不限，内边距 24px，背景 `--surface-base`

### 3.2 列表页标准结构

```vue
<div class="page-header">       <!-- 标题 + 描述 -->
<div class="toolbar">           <!-- 左侧筛选 / 右侧操作按钮 -->
<div class="table-container">   <!-- 表格，与工具栏无缝衔接 -->
```

- 工具栏与表格共用外边框，消除「双边框」视觉
- 操作按钮右对齐，间距 8px
- 表格操作列固定右侧（`:fixed="'right'"`）
- 工具栏筛选用 **默认 32px** 控件，与页面表单一致

### 3.3 弹窗 / 抽屉表单结构

```vue
<a-modal>
  <div class="form-controls-md">
    <!-- 第一区：基础信息，a-row :gutter="16" class="mb-4" -->
    <!-- 第二区：分栏编辑器（如 builder-split） -->
  </div>
</a-modal>
```

- 弹窗内所有输入类控件统一 `form-controls-md`
- 分区之间 `margin-bottom: 16px`
- 分栏面板内用 `field-stack` 组织筛选字段

### 3.4 响应式断点

| 断点 | 宽度 | 布局策略 |
|------|------|----------|
| sm | < 768px | 侧栏 overlay，单列，表单垂直排列 |
| md | 768–1024px | 侧栏可折叠，双列卡片 |
| lg | > 1024px | 完整侧栏，三列及以上 Grid |

## 4. 组件规范

### 4.1 图标

- **库**：`@arco-design/web-vue/es/icon`，禁止混用其他图标库
- **用法**：

```javascript
import { IconApps, IconLink, IconSettings } from '@arco-design/web-vue/es/icon'
```

```vue
<icon-apps :size="18" />
```

| 场景 | 尺寸 |
|------|------|
| 按钮内 | 14px |
| 导航 / 列表 | 16–18px |
| 区块标题 | 20px |
| 空状态 | 48px |

颜色跟随 `currentColor`，不写死 hex 值。

### 4.2 按钮

| 类型 | size | 高度 | 样式 |
|------|------|------|------|
| Primary | 默认 | 32px | 背景 `#0ea5e9`，文字白色，`border-radius: 6px` |
| Secondary | 默认 | 32px | 白底 + `#e2e8f0` 边框，悬停 `#f8fafc` 背景 |
| Ghost / Text | `small` | 28px | 无边框，悬停轻背景 |
| Icon Only | 默认 | 32×32px | 居中图标，悬停圆角背景 |
| 行内操作 | `mini` | 24px | 表格/积木块内 ↑↓删除 |

- Focus：边框变 Primary，**不使用** ring / box-shadow 光晕（见 `globals.css`）
- 危险操作：文字/边框 `#dc2626`，禁止用 Primary 色
- 禁用：`opacity: 0.5`，不可点击
- **与输入框同行时**：按钮 `size` 须与输入框对齐（同为默认或同为 `small`）

### 4.3 表单（详细规范）

共享样式文件：`ui/src/styles/form-controls.css`（全局引入）

#### 4.3.1 布局模式

**模式 A — 页面顶栏表单（多列）**

```vue
<a-row :gutter="16" class="mb-4">
  <a-col :span="8">
    <div class="field-label">设备 ID <span class="req">*</span></div>
    <a-input v-model="form.id" />
  </a-col>
</a-row>
```

- 列间距 `gutter="16"`
- 行间距 `mb-4`（16px）
- 标签与控件间距 6px

**模式 B — 字段堆叠（侧栏 / 筛选区）**

```vue
<div class="field-stack">
  <div class="form-field">
    <div class="field-label-sm">源设备通道</div>
    <a-select v-model="channelId" />
  </div>
  <div class="form-field">
    <div class="field-label-sm">筛选设备</div>
    <a-input-search search-button />
  </div>
</div>
```

- 字段之间 `gap: 12px`（`.field-stack`）
- 标签与控件 `gap: 6px`（`.form-field`）
- 堆叠区与下方列表 `margin-bottom: 12px`

**模式 C — Arco Form 水平布局**

```vue
<a-form class="industrial-form form-controls-md" layout="horizontal"
  :label-col-props="{ span: 6 }" :wrapper-col-props="{ span: 18 }">
```

- 表单项间距 14px
- 标签 13px / 500 / `#475569`

#### 4.3.2 输入框视觉规格（Medium / 默认）

| 属性 | 值 |
|------|-----|
| 高度 | 32px（`min-height`） |
| 圆角 | 6px |
| 背景 | `#fafafa`（Light）/ `#334155`（Dark） |
| 边框 | 1px `#e2e8f0` |
| Focus 边框 | `#0ea5e9`，无 glow |
| 字号 | 14px |
| 内边距 | 左右 12px |

#### 4.3.3 各控件类型规范

| 控件 | size | 特殊说明 |
|------|------|----------|
| `a-input` | 默认 | 单行文本 |
| `a-input-search` | 默认 | `search-button` 时按钮高度须与输入框一致 |
| `a-select` | 默认 | 与同级 input 同高；`allow-search` 不改变高度 |
| `a-input-number` | 默认 | 与 input 同宽策略 |
| `a-textarea` | 默认 | 用 `:auto-size="{ minRows: 2, maxRows: 6 }"`，不设 `size="small"` |
| `a-switch` | — | 独立组件，标签用 `.field-label` 水平排列 |
| `a-checkbox` / `a-radio` | — | 列表多选场景；按钮组用 `size="mini"` |

#### 4.3.4 容器 class 对照

| Class | 作用 |
|-------|------|
| `.form-controls-md` | 强制区域内控件 32px、圆角 6px、搜索按钮对齐 |
| `.form-controls-sm` | 紧凑区 28px（表格内嵌整组使用） |
| `.field-stack` | 字段纵向排列，间距 12px |
| `.form-field` | 单字段：标签 + 控件，间距 6px |
| `.field-label` / `.field-label-sm` | 12px 标签，不换行 |
| `.industrial-form` | Arco Form 表单项间距与标签样式 |
| `.mb-1` ~ `.mb-6` | 4px 递增间距工具类 |

#### 4.3.5 分组标题

`.nb-form-section__title`：13px / 600 / `#475569`，底部虚线 `#e2e8f0`，与下方字段间距 12px。

#### 4.3.6 校验与错误

- 错误态：`:status="'error'"` + `.field-error` 11px 红色说明
- 错误说明距输入框 4px
- 必填：标签后红色 `*`（`.req`）

### 4.4 表格

工业数据表格保持**高密度、低装饰**：

| 属性 | 值 |
|------|-----|
| 行高 | 40px |
| 表头背景 | `#f1f5f9` |
| 表头文字 | 12px / 600 / `#64748b` |
| 单元格 padding | `0 12px` |
| 边框 | 行底 1px `#e2e8f0` |
| 悬停行 | 背景 `#f8fafc` |
| 选中行 | 左 2px `#0ea5e9` 边条 + `rgba(14,165,233,0.05)` 背景 |
| 圆角 | 0（表格主体），容器外框可用 6px |
| 内嵌编辑控件 | 统一 `size="small"` + `.form-controls-sm` |

**内容规则**：
- 左对齐；状态列用 `<a-tag>` 或自定义 Badge

全局样式见 `ui/src/styles/globals.css`。

### 4.5 卡片

| 属性 | 值 |

### 4.7 弹窗与抽屉

- 圆角 8px，标题 16px 600
- 内容区最大高度 `70vh`，超出滚动
- 根容器加 `form-controls-md` 保证输入高度统一
- 底部操作栏：右对齐，主按钮在右，取消在左
- 模式提示 Banner（如北向 Push/Passive）：见 `northbound-form.css`

### 4.8 通知与反馈

- 全局消息：右上角 `a-notification`，3s 自动关闭
- 加载：表格/列表用 Skeleton；按钮内用 `loading` 态
- 页面切换：`fade` 过渡 200ms
- 错误：表单字段下方红色 12px 说明 + 顶部 Notification

## 5. 交互规范

| 场景 | 规范 |
|------|------|
| 悬停 | 背景/边框微变，150ms ease |
| 点击 | 按钮 `:active` 轻微 scale 或加深背景 |
| Focus | 边框 Primary，无 glow |
| 禁用 | opacity 0.5，cursor not-allowed |
| 空状态 | 48px 图标 + 一行说明 + 可选主操作按钮 |
| 确认删除 | Modal 二次确认，危险按钮红色 |

## 6. 深色模式

所有新增页面/组件必须测试 Dark 主题。约定：

1. 背景、边框、文字使用 Arco CSS 变量或项目 `--surface-*` / `--text-*`
2. 卡片/表格/输入框跟随 `.dark-theme` 下的全局覆盖（见 `App.vue`）
3. 功能色（Success/Warning/Error）深浅主题保持一致
4. 截图验收：每个主要页面需提供 Light + Dark 截图

## 7. 性能与工程

- **组件复用**：MetricCard、EdgeProtocolBadge、StatusIndicator 等公共组件优先引用
- **样式组织**：
  - 全局：`ui/src/styles/globals.css`（表格、Focus）
  - 表单：`ui/src/styles/form-controls.css`（尺寸、间距、字段堆叠）
  - 业务：`ui/src/styles/northbound-form.css`（北向弹窗 Banner 等）
  - 页面：scoped CSS 仅放页面特有样式
- **Tailwind**：仅用于原子类（flex、gap、truncate），复杂组件用 CSS 类
- **懒加载**：路由级 code split；ECharts 等大型依赖动态 import
- **图标**：统一 Arco Icon 组件，避免内联 SVG 重复

## 8. 浏览器支持

- **推荐**：Chrome、Edge、Firefox、Safari 最新两个大版本
- **不支持**：IE11（已移除兼容代码）

## 9. 验收清单

优化或新增页面时，对照以下标准：

- [ ] 符合 8px 栅格间距，字段之间不贴边（`field-stack` ≥ 12px）
- [ ] **同一表单区域输入控件高度统一**（默认 32px，无 default/small 混用）
- [ ] `a-input-search` 搜索按钮与输入框等高
- [ ] 间距工具类（`mb-*`）已定义或使用 `field-stack` / `form-field`
- [ ] 色彩克制，功能色仅用于状态/协议/主操作
- [ ] 表格、表单、卡片与本文档规格一致
- [ ] Light / Dark 主题均可正常使用
- [ ] 交互有明确反馈（加载、成功、错误）
- [ ] 1280px 与 768px 下布局正常
- [ ] 无多余边框、双边框、未使用的阴影
