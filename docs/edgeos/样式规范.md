# EdgeOS 工业大脑 UI 开发实操指南 (基于 v4.1 标准)

## 文档定位

本文档是 EdgeOS UI 设计规范的 **工程化落地版本**，从"视觉指南"进化为 **"工业级工程标准"**。它针对 EdgeOS 这种高实时性、高可靠性的边缘计算系统，在 **shadcn/ui (Radix UI)** 的基础上进行了深度"降噪"处理，确保界面在极端工业环境下（如高温、震动、弱光、触控手套操作）依然保持高可读性和操作精确度。

**v4.1 升级说明**：v4.1 是在 v4.0 工业约束风格之上的 **Modern Refactor Overlay**——不推翻直角、高密度、强语义等工业基因，而是通过 **间距系统重构 + 降低视觉噪声 + 重分配层级节奏**，实现"现代工业 SaaS"的可扫读性与呼吸感（类似 Grafana + Linear 混合风格）。

---

## 0. v4.1 核心变化原则

从 v4.0 → v4.1 的变化不是"风格改变"，而是三件事：

| 维度 | v4.0（旧模型） | v4.1（新模型） |
| :--- | :--- | :--- |
| 结构定义 | 以 **边框** 定义结构 | 以 **空间（spacing）** 定义结构 |
| 信息密度 | 以"紧凑密度"提升信息量 | 以 **层级间距（hierarchy gap）** 替代边框分割 |
| 边框使用 | 组件之间靠 1px border 分割 | 仅在 **语义断层** 使用 border |

### 0.1 最关键的三条现代化约束

如果只做三件事，就做这三条：

1. **去 border 化（最重要）**：border 只用于分区、卡片边界；不用于结构分割
2. **增加垂直空间（核心体验变化）**：`section-gap: 24px → 32px`，`page-padding: 16px → 24px`
3. **减少视觉噪声**：降低 hover 强度、降低 border contrast、减少全局 `slate-200` 使用频率

### 0.2 呼吸感公式

所有组件必须满足：

```
视觉节奏 = 内容高度 : 空间间隔 ≥ 1 : 0.6
```

| 内容高度 | 建议间距 |
| :--- | :--- |
| 40px row | 24px gap |
| 44px row | 24~32px gap |
| card | 24~32px gap |

## 0.3 间距 Token 系统（必须替换零散 px/py）

统一引入 **8pt + 呼吸倍率** 体系，禁止在组件中随意写 magic number。

| Token | 值 | Tailwind 等价 | 用途 |
| :--- | :--- | :--- | :--- |
| `space-1` | 4px | `p-1` / `gap-1` | 微调 |
| `space-2` | 8px | `p-2` / `gap-2` | 元素内间距 |
| `space-3` | 12px | `p-3` / `gap-3` | 紧凑组件 gap |
| `space-4` | 16px | `p-4` / `gap-4` | 标准组件 padding |
| `space-6` | 24px | `p-6` / `gap-6` | 模块间距（核心） |
| `space-8` | 32px | `p-8` / `gap-8` | 区块间距（强呼吸） |
| `space-12` | 48px | `p-12` / `gap-12` | 页面分区 |

**CSS 变量定义**（`globals.css`）：

```css
:root {
  --space-1: 4px;
  --space-2: 8px;
  --space-3: 12px;
  --space-4: 16px;
  --space-6: 24px;
  --space-8: 32px;
  --space-12: 48px;

  /* 语义别名 */
  --page-padding-x: var(--space-6);
  --page-padding-bottom: var(--space-8);
  --section-gap: var(--space-6);
  --card-gap: var(--space-6);
  --card-padding: var(--space-6);
}
```

| :--- | :--- | :--- |

## 1. 核心布局架构 (Layout Implementation)

在工业场景中，侧边栏（Sidebar）不仅仅是导航，更是系统状态的常驻监控区。布局必须保持稳定，避免因内容滚动导致的界面晃动。

**v4.1 布局核心**：页面内容 **不再贴边**，模块之间默认 `24px gap`，卡片之间至少 `16px gap`。

### 1.1 整体布局结构

```
┌─────────────────────────────────────────────────────────────────────────────┐
│                              App Container                                   │
│  ┌──────────────┬─────────────────────────────────────────────────────────┐ │
│  │              │  Header (64px)                                           │ │
│  │   Sidebar    │  ┌─────────────────────────────────────────────────────┐ │ │
│  │   (240px)    │  │ 面包屑 / 当前页面          [全局搜索] [用户] [设置] │ │ │
│  │              │  └─────────────────────────────────────────────────────┘ │ │
│  │  ┌────────┐  │                                                         │ │
│  │  │ Logo   │  │  Main Content (动态路由)                                 │ │
│  │  └────────┘  │  ┌─────────────────────────────────────────────────────┐ │ │
│  │   ↕ 16px     │  │                                                     │ │ │
│  │  Nav Group 1 │  │  Page Container                                      │ │ │
│  │   ↕ 24px     │  │  padding: 24px 24px 32px 24px                      │ │ │
│  │  Nav Group 2 │  │  gap: 24px (模块间距)                               │ │ │
│  │   ↕ auto     │  │                                                     │ │ │
│  │  Status Area │  │  [页面内容 — 悬浮内容区，不贴边]                     │ │ │
│  │              │  │                                                     │ │ │
│  └──────────────┴─────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────────────────────┘
```

**Page Container 规范**：

```css
/* 页面内容区 — 悬浮布局，不贴边 */
.page-container {
  padding: 24px 24px 32px 24px;  /* space-6 / space-6 / space-8 / space-6 */
  gap: 24px;                        /* 模块默认间距 */
  display: flex;
  flex-direction: column;
}
```

### 1.3 页头 (Header) 实现规范

v4.1 核心变化：减少"盒子感"，增加呼吸空间。

| 属性 | v4.0 | v4.1 | 说明 |
| :--- | :--- | :--- | :--- |
| 高度 | `56px` (`h-14`) | `64px` (`h-16`) | 增加垂直呼吸 |
| 背景 | `bg-white` | `bg-white` | 纯白背景 |
| 下边框 | `border-b border-slate-200` | `border-b border-slate-100` | 更轻的分割线 |
| 内边距 | `px-6` | `px-8` | 左右 32px |
| 布局 | `flex items-center justify-between` | 不变 | 左右分布 |

**面包屑规范**：

- 强制显示完整路径，不省略中间层级
- 分隔符统一使用 **斜杠 `/`**（非 `>` 或 `»`）
- 当前页面不可点击，文字色 `text-slate-900 font-medium`（L1）
- 上级路径可点击，文字色 `text-slate-500 hover:text-slate-900 font-normal`（L2）
- 项间距 `gap-3`（12px），`letter-spacing: 0.2px`

**全局搜索 (Command Palette)**：

- 触发方式：点击搜索框或快捷键 `Ctrl+K` (Mac: `Cmd+K`)
- 搜索范围：设备、协议、配置文件、文档
- 使用 shadcn `Command` 组件实现
- 搜索框占位符：`搜索设备、协议或文档... (Ctrl+K)`

## 2. 工业级数据表 (Industrial Data Table)

表格是 EdgeOS 的资产管理核心，必须处理每页 50-100 行的高密度数据，支持虚拟滚动以优化性能。

v4.1 核心变化：轻压缩视觉优化——降低 border 对比度，增加行高，减轻 hover 强度。

### 2.1 行高密度样式

| 模式 | 行高 | v4.1 变化 | 适用场景 |
| :--- | :--- | :--- | :--- |
| 常规模式 | `h-11` (44px) | 原 40px → 44px | **默认展示**，兼顾可读性与密度 |
| 紧凑模式 | `h-8` (32px) | 不变 | 监控大屏、高级用户、高密度数据展示 |
| 宽松模式 | `h-12` (48px) | 不变 | 触控屏、工业手套操作环境 |

## 3. 实时监控组件 (Monitoring Components)

### 3.1 状态指示器 (The "Pulse" Standard)

工业状态不仅靠颜色，还要靠"动态感知"，让操作人员在余光中也能感知状态变化。

**色彩语义**：

| 状态 | 颜色 | 动画 | 语义 |
| :--- | :--- | :--- | :--- |
| **运行中** | `emerald-500` | `animate-pulse` | 系统正常、逻辑通过、在线 |
| **待机/离线** | `amber-500` | 无 | 离线/待机、非致命警告、连接中断 |
| **故障** | `red-500` | `animate-pulse` | 指令超时、硬件故障、通讯中断 |
| **未知** | `slate-400` | 无 | 状态未上报、初始化中 |

**双层脉冲动画实现**：

```vue
<!-- components/edge/StatusIndicator.vue -->
<template>
  <div class="relative flex h-2 w-2">
    <!-- 外层脉冲环 (仅运行和故障状态显示) -->
    <span
      v-if="showPulse"
      class="absolute inline-flex h-full w-full animate-ping rounded-full opacity-75"
      :class="pulseColorClass"
    ></span>
    <!-- 内层实心点 -->
    <span
      class="relative inline-flex h-2 w-2 rounded-full"
      :class="dotColorClass"
    ></span>
  </div>
</template>

<script setup lang="ts">
const props = defineProps<{
  status: 'running' | 'standby' | 'fault' | 'unknown'
}>()

const showPulse = computed(() => props.status === 'running' || props.status === 'fault')

const dotColorClass = computed(() => {
  switch (props.status) {
    case 'running': return 'bg-emerald-500'
    case 'standby': return 'bg-amber-500'
    case 'fault': return 'bg-red-500'
    default: return 'bg-slate-400'
  }
})

const pulseColorClass = computed(() => {
  switch (props.status) {
    case 'running': return 'bg-emerald-400'
    case 'fault': return 'bg-red-400'
    default: return ''
  }
})
</script>
```

### 3.2 协议徽章 (Protocol Badges)

统一封装 `<EdgeProtocolBadge protocol="modbus" />`，根据协议类型自动匹配视觉样式。

| 协议 | 背景色 | 文字色 | 说明 |
| :--- | :--- | :--- | :--- |
| Modbus/TCP | `bg-sky-500` | `white` | 科技蓝，最常用 |
| Modbus/RTU | `bg-sky-600` | `white` | 深蓝，串口版本 |
| OPC UA | `bg-indigo-500` | `white` | 深靛蓝 |
| MQTT | `bg-emerald-500` | `white` | 翠绿色 |

### 4.3 防抖指令封装

工业协议搜索、参数下发指令必须经过防抖处理，避免频繁请求。

```typescript
// composables/useDebounce.ts
export function useDebounce<T extends (...args: any[]) => any>(
  fn: T,
  delay: number = 300
) {
  let timer: NodeJS.Timeout | null = null

  return (...args: Parameters<T>) => {
    if (timer) clearTimeout(timer)
    timer = setTimeout(() => fn(...args), delay)
  }
}

// 使用示例
const searchDevices = useDebounce((keyword: string) => {
  api.searchDevices(keyword)
}, 300)
```

### 4.4 节流指令封装

用于滚动加载、窗口 resize 等高频事件。

```typescript
// composables/useThrottle.ts
export function useThrottle<T extends (...args: any[]) => any>(
  fn: T,
  delay: number = 200
) {
  let lastCall = 0

  return (...args: Parameters<T>) => {
    const now = Date.now()
    if (now - lastCall >= delay) {
      lastCall = now
      fn(...args)
    }
  }
}
```
