## 存储组件

### 1. 配置存储 (`internal/sync/config_store.go`)
- **用途**: 管理带有 WAL、内存缓存和磁盘持久化的配置记录
- **核心特性**:
  - 预写日志 (WAL) 保证数据完整性
  - 内存缓存提供快速读取操作
  - 按绑定键 (BindingKey) 索引以提高查找效率
  - 支持多种桶类型

### 2. 存储后端 (`internal/storage/boltdb.go`)
- **用途**: BoltDB 原始操作，用于时间序列和缓存值
- **核心特性**:
  - 通用桶操作
  - 范围查询和迭代
  - 数据修剪和保留管理

### 3. 同步管理器 (`internal/sync/sync_manager.go`)
- **用途**: 协调分布式节点间的同步
- **核心特性**:
  - Gossip 协议实现点对点通信
  - 基于反熵摘要的同步机制
  - 设备接管和状态管理

## 桶策略

| 桶类型 | 模式 | 用途 | 保留期 | 最大记录数 |
|--------|------|------|--------|------------|
| device | `device_{channelID}` | 按通道的设备数据存储 | 30 天 | 1,000,000 |
| snapshot | `snapshot_{deviceID}` | 设备状态快照 | 7 天 | 1,000 |
| sync_node | `sync_node_{nodeID}` | 同步节点元数据 | 90 天 | 10,000 |
| values | `values` | 最新值缓存 | 7 天 | 500,000 |
| wal | `wal` | 预写日志 | 1 天 | 100,000 |
| config | `config` | 配置记录 | 365 天 | 10,000 |

## 分区方案

### 主键格式
```

{deviceID}_{timestamp_hour}
```
### 次键格式
```

{deviceID}_{unix_timestamp}_{sequence}
```
### 键示例
```

slave-1_1699584000          // 主分区键（设备 + 小时）
slave-1_1699584001_000000000001  // 带序列号的次键
```
### 时间粒度
- **主分区**: 小时级精度
- **次索引**: 毫秒级精度

## 数据流

### 写入流程
1. 从设备采集数据点
2. 添加到内存数据管道
3. 根据通道/设备 ID 保存到相应的桶
4. 写入 WAL 条目以确保持久性
5. 更新内存索引
6. （可选）向对等节点广播同步消息

### 读取流程
1. 接收查询请求
2. 首先检查内存缓存
3. 如果未找到，执行磁盘查找
4. 返回结果（可选聚合）

### 同步流程
1. 对等节点间周期性交换摘要
2. 基于版本比较计算增量
3. 从对等节点拉取缺失的记录
4. 向对等节点通告新记录
5. 使用向量时钟解决冲突

## 冲突解决

### 向量时钟策略
- 每条记录都有一个版本号
- 每次更新时版本号递增
- 检测到冲突时：
  - 版本号高的获胜
  - 版本号相同：最新时间戳获胜
  - 时间戳相同：比较节点 ID

### 反熵协议
- 周期性摘要交换
- 基于增量的同步
- 按需全量同步

## 性能优化

### BoltDB 调优
- **页大小**: 64KB（针对顺序写入优化）
- **缓存大小**: 128MB（减少磁盘 I/O）
- **超时时间**: 30 秒（处理慢速磁盘场景）
- **写缓冲区**: 64KB（高效批量写入）

### 分区优化
- 设备级分区减小桶大小
- 小时级分区实现高效修剪
- 基于前缀的迭代加速范围查询

### 并发控制
- **最大并发写入**: 1000
- **最大并发读取**: 500
- **事务超时**: 5 秒

## 监控与告警

### 指标
- 磁盘使用率百分比
- 写入延迟（目标：<100ms）
- 读取延迟（目标：<50ms）
- 活跃连接数

### 告警阈值
- 磁盘使用率：90%
- 写入延迟：100ms
- 读取延迟：50ms

## 最佳实践

1. **桶命名**: 遵循 `device_{channelID}` 模式存储设备数据
2. **键设计**: 使用时间戳键存储时间序列数据
3. **保留策略**: 为每种桶类型配置适当的保留期
4. **监控**: 启用指标以早期发现问题
5. **备份**: 定期备份关键配置数据

## 故障恢复

1. **WAL 恢复**: 启动时重放 WAL 条目
2. **内存重建**: 从磁盘加载配置记录到内存
3. **对等同步**: 恢复后与对等节点同步以赶上进度
4. **快照恢复**: 使用快照进行时间点恢复

## 安全考虑

1. **文件权限**: 数据库文件限制为所有者访问（0600）
2. **访问控制**: 桶级访问模式配置
3. **数据完整性**: 使用 SHA256 哈希验证记录
4. **加密**: 敏感数据考虑静态加密
