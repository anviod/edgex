工业协议网关与边缘计算运行时

单进程运行时，南向协议适配器接入，ShadowCore 内存状态持久化，边缘规则执行，北向数据转发。支持 Linux、Windows 与 ARM64 网关；无需云端依赖，无需容器。

系统定位

面向现场设计的边缘运行时

确定性采集、本地状态持久化、协议无关的北向分发

EdgeX 以单进程运行，将南向协议适配器绑定到统一的通道/设备模型，实时状态写入 ShadowCore，由 Execution Mapper 执行边缘规则，并通过 MQTT、HTTP、OPC UA Server、BACnet Server、Sparkplug B 或数据库连接器转发数据。配置采用声明式且跨重启持久化；运行时数据可重建，不影响配置库。

核心子系统

运行时组件

Protocol Layer

协议适配器

13 个南向适配器：Modbus RTU/TCP、BACnet IP、OPC UA Client、Siemens S7、EtherNet/IP、IEC 104、DLT645-2007、KNXnet/IP、SNMP v2c/v3、Omron FINS、Mitsubishi SLMP、Profinet IO、EtherCAT M1。

Scheduler

ScanEngine

基于 EDF 的扫描调度器，自适应节流、RTT/MTU 管理器、熔断与设备级隔离。目标：10k 点位 lag P95 <100 ms。

State

ShadowCore

点位值的内存事实来源，支持写时复制快照、环形缓冲区写入和分钟级持久化到 runtime.db。

Compute

Execution Mapper

边缘规则引擎，支持表达式求值、预处理流水线、批量动作与事件记录。本地执行，无需回环云端。

Virtualization

Virtual Shadow

跨通道引用计算点和聚合点，公式依赖图与跨设备聚合。

Northbound

北向管理器

统一发布器，支持 store-and-forward、自动重连、主题模板与负载 schema 控制。

工程准则

设计原则

  • 稳定性优先于过早优化
  • 简单性优先于投机式抽象
  • 本地执行优先于云端依赖
  • 可恢复性优先于峰值吞吐
  • 统一模型优先于功能堆砌

所有变更在真实 ARM64 硬件上测量，并通过 soak 回归验证后方可合并。

执行层

ScanEngine + Execution Mapper

确定性扫描调度与本地规则执行

ScanEngine 将点位分配到扫描类，应用 EDF 调度，通过 RTT/MTU 反馈自适应批量大小，并隔离故障设备以避免级联延迟。Execution Mapper 针对 ShadowCore 快照求值规则，触发动作并将事件记录到 WAL。两个子系统均为单二进制、零依赖，并可通过指标端点观测。
