### 10.4 与南向采集的关系

- **ScanEngine 采集路径**：驱动 `ReadPoints` → ChannelManager → `ShadowIngress.IngestDirect` → ring buffer → 定时/满缓冲 `ApplyShadowWrites`（Q4 批量写，降低锁竞争）。
- **REST/写点路径**：仍直写 `ShadowCore.WriteShadowDevice`（低延迟，不经 Ingress 缓冲）。
- **ConnectionManager**：各协议 transport（Modbus / S7 / DL/T645 等）共享退避/Dead/半开逻辑；scenario 测试在子包中验证集成行为，主包单测保证公共组件 ≥80% 覆盖。

### 10.5 bufferReliable 注入与单测（2026-06-30 晚）

**问题**：QoS≥1 直写失败时需进入 `bufferReliable` → `replayReliable`，但 `WriteShadowDevice` 正常路径不返回 error，无法覆盖。

**方案**：在 `ShadowIngress` 增加可选 `writeShadow` 函数字段（默认 `nil`，生产走 `shadowCore.WriteShadowDevice`）；单测同包直接赋值模拟失败。

| 测试函数 | 覆盖路径 |
| :--- | :--- |
| `TestShadowIngress_BufferReliable_IngestQoS1` | `Ingest` QoS≥1 写失败 → `bufferReliable` → 手动 `replayReliable` 成功 |
| `TestShadowIngress_BufferReliable_IngestDirect` | `IngestDirect` QoS≥1 写失败 → replay |
| `TestShadowIngress_ReplayReliable_OnStart` | 失败缓冲后 `Start()` 触发 `replayReliable` |
| `TestShadowIngress_BufferReliable_ReplayStillFails` | replay 仍失败时消息留 pending、影子设备未创建 |

`bufferReliable` / `replayReliable` / `writeShadowDevice` 语句覆盖率：**100%**。
