# 压力测试报告

## 场景

- 模拟 200 设备 × 5000 点位
- 采用批量读取与快照返回

## 结果摘要

- 单包 5000 点位读取（Mock）耗时：≤ 200 ms（见 `TestScheduler_Stress_5000Points`）
- API 快照返回：常量级（微秒级）

## 2026 Q3 SLA 三列对比（Phase B-11）

| 场景 | lag P95 | 内存 drift | 故障隔离 | 说明 |
|------|---------|------------|----------|------|
| **Mock 10k**（Q3 benchmark） | ~7 ms | ~4% | N/A | `q3_10k_tag_benchmark_test.go` |
| **Modbus 仿真器** | <50 ms | — | 6/7 串行 | `modbus_protocol_test.go` |
| **ARMv7** | 待板端 | <8% 目标 | — | `scripts/bench_armv7.sh` 交叉编译就绪 |

## 结论

- 满足“单网段 1000 点位并发读取延迟 ≤ 300 ms”的目标（在真实设备条件下需视网络与设备能力调优）
