# 一、BACnet objectList 扫描 测试目标

在点击“扫描对象”操作时，确保系统能够：

1. **完整扫描设备的 objectList**
2. **准确区分新对象、已有对象和已删除对象**
3. **保证扫描结果与设备实际对象状态一致**
4. **在异常场景下保持系统稳定与数据一致性**

---

# 二、测试范围

* 设备对象扫描功能（Scan Objects）
* 新旧对象差异识别机制（Diff）
* 对象状态标记（new / existing / removed）
* 扫描结果的稳定性、准确性与可追溯性

---

# 三、测试举证（测试方案）

## 1️⃣ 功能正确性测试

### 场景 OBJ-FUNC-01：首次扫描（冷启动）

**前置条件：**
系统无该设备的历史对象记录。

**操作：**
点击“扫描对象”。

**期望结果：**

* 返回对象数量与设备 objectList 一致（如日志中为 13 个）。
* 所有对象均标记为：`new`。

---

### 场景 OBJ-FUNC-02：二次扫描（无变化）

**前置条件：**
设备对象未发生变化。

**操作：**
再次点击“扫描对象”。

**期望结果：**

* 返回对象数量保持不变。
* 所有对象标记为：`existing`。
* 无对象标记为 `new` 或 `removed`。

---

### 场景 OBJ-FUNC-03：新增对象

**前置条件：**
设备新增 1~N 个对象。

**操作：**
点击“扫描对象”。

**期望结果：**

* 新增对象标记为：`new`。
* 原有对象标记为：`existing`。
* 无对象误判为 `removed`。

---

### 场景 OBJ-FUNC-04：删除对象

**前置条件：**
设备删除 1~N 个对象。

**操作：**
点击“扫描对象”。

**期望结果：**

* 缺失对象标记为：`removed`。
* 仍存在对象标记为：`existing`。
* 无对象误判为 `new`。

---

### 场景 OBJ-FUNC-05：混合变化（新增 + 删除）

**前置条件：**
设备同时新增和删除对象。

**操作：**
点击“扫描对象”。

**期望结果：**

* 新增对象 → `new`
* 未变对象 → `existing`
* 删除对象 → `removed`

---

## 2️⃣ 数据完整性与一致性测试

### 场景 OBJ-DATA-01：对象唯一性验证

**操作：**
扫描包含多个对象。

**期望结果：**

* 不存在 Type+Instance 重复对象。
* 重复对象应被合并或标记异常。

---

### 场景 OBJ-DATA-02：字段完整性验证

**操作：**
扫描对象列表。

**期望结果：**
每个对象至少包含：

* Object Type
* Object Instance
* 状态标记（new/existing/removed）
* 最近扫描时间戳

---

### 场景 OBJ-DATA-03：跨扫描一致性

**操作：**
多次连续扫描同一设备。

**期望结果：**

* 对象标识稳定不漂移。
* 状态转换逻辑符合实际变化。

---

## 3️⃣ 异常与边界场景测试

### 场景 OBJ-ERR-01：设备返回空 objectList

**操作：**
点击“扫描对象”。

**期望结果：**

* 所有历史对象标记为：`removed`。
* 系统不崩溃，不报致命错误。

---

### 场景 OBJ-ERR-02：扫描超时或通信失败

**操作：**
设备无响应或通信中断。

**期望结果：**

* 保留上一次有效对象数据。
* 返回扫描失败状态并记录日志。
* 不误修改对象状态。

---

### 场景 OBJ-ERR-03：对象数据格式异常

**操作：**
设备返回非法或畸形对象标识。

**期望结果：**

* 丢弃异常对象。
* 正常对象不受影响。
* 系统不 panic。

---

### 场景 OBJ-ERR-04：短时间多次扫描

**操作：**
连续多次点击“扫描对象”。

**期望结果：**

* 系统处理有序。
* 不出现数据错乱、重复记录或状态错乱。

---

## 4️⃣ 用户界面与结果呈现测试

### 场景 OBJ-UI-01：新对象高亮显示

**期望结果：**

* `new` 对象在 UI 中明显标识（如高亮、图标）。

---

### 场景 OBJ-UI-02：删除对象可追溯

**期望结果：**

* `removed` 对象可查看历史记录或变更说明。
* 不直接静默消失（除非配置允许）。

---

### 场景 OBJ-UI-03：扫描结果可导出

**期望结果：**

* 扫描结果可导出为 JSON/CSV/报表（若产品支持）。

# 四、验证标准（Acceptance Criteria）

## 1️⃣ 功能准确性标准

| 项目      | 合格标准                   |
| ------- | ---------------------- |
| 对象扫描完整率 | 100% 与设备 objectList 一致 |
| 新旧区分正确率 | 100%                   |
| 删除识别正确率 | 100%                   |
| 重复对象识别率 | 100%                   |

---

## 2️⃣ 数据一致性标准

| 项目     | 合格标准              |
| ------ | ----------------- |
| 对象唯一性  | 无重复 Type+Instance |
| 状态稳定性  | 无无故状态跳变           |
| 跨扫描一致性 | 同对象标识保持稳定         |

---

## 3️⃣ 稳定性与鲁棒性标准

| 项目     | 合格标准         |
| ------ | ------------ |
| 异常输入处理 | 不 panic、不崩溃  |
| 通信异常处理 | 保留旧数据，提示失败   |
| 高频扫描处理 | 不造成数据错乱或系统阻塞 |

---

## 4️⃣ 性能标准

| 项目      | 合格标准                 |
| ------- | -------------------- |
| 扫描响应时间  | ≤ 2 秒（约 10–20 个对象规模） |
| CPU 使用率 | 扫描期间 ≤ 10%           |
| 内存增长    | 无持续增长（无泄漏）           |

---

## 5️⃣ 可追溯性与审计标准

| 项目     | 合格标准            |
| ------ | --------------- |
| 扫描日志   | 记录扫描时间、设备ID、对象数 |
| 变更日志   | 记录新增/删除对象详情     |
| 审计可追溯性 | 可回溯对象变化历史       |

---

# 五、验收判定规则

系统通过对象扫描功能验收，必须满足：

1. 所有功能测试项：**100% 通过**
2. 所有新旧区分场景：**无误判**
3. 所有异常场景：**无崩溃、无数据破坏**
4. 所有性能指标：**满足标准**
